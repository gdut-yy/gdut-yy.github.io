    武松本人不好意思承认，便偷偷找了鲁智深商量对策。

    鲁智深听完，哈哈大笑：“武二郎啊武二郎，你可真行，打了只病猫还到处吹牛。人家猫可有 999 条命，你也就打掉 111 条，剩下的命可还多着呢！”

    现在，请你帮武松算算，那只病猫剩下的命数是多少。

    输入格式

    无。

    输出格式

    输出一个整数，表示答案。

    T2. 情报传递 1【算法赛】

    问题描述

    请注意：本题与情报传递 222 的区别仅在于测试数据数量与数据范围不同。

    在北宋末年的乱世，梁山泊的好汉们经常要穿梭于各地，传递情报和策划行动。一天，智多星吴用接到了一项紧急任务，需要派遣鲁智深从一个编号为 aaa 的城市出发，前往编号为 bbb 的城市，以商讨与盟友的下一步行动。

    然而，朝廷已经在沿途布下了天罗地网，凡是编号为 ccc 的倍数的城市都设有重重哨卡，一旦经过这些城市，梁山的行踪就会被朝廷发现，后果不堪设想。因此，鲁智深在赶往 bbb 号城市的路上，必须避开这些编号为 ccc 倍数的城市。

    每次，鲁智深可以选择沿着大道前进，假设当前鲁智深在 xxx 号城市，那么经过一天徒步他可以到达 x+1x+1x+1 或 x+2x+2x+2 号城市 ，但无论如何都不能进入那些危险的城市。为了万无一失，吴用需要设计出一条最安全且最短的路径，让鲁智深能以最少的步数到达 bbb 号城市。

    现在，请你帮助吴用计算：鲁智深最少需要多少天，才能从 aaa 号城市安全抵达 bbb 号城市？

    输入格式

    第一行输入一个整数 t(t=1)t(t=1)t(t=1) 表示测试数据的数量。

    接下来 ttt 行，每行包括三个整数 a,b,c(1≤a<b≤105,2≤c≤105)a,b,c(1 \leq a <b \leq 10^5,2 \leq c \leq 10^5)a,b,c(1≤a<b≤105,2≤c≤105) 表示一组测试数据。

    数据保证 a,ba,ba,b 不为 ccc 的倍数。

    输出格式

    输出 ttt 行，每行一个整数表示答案。

    输入样例

    1
2 7 3

    输出样例

    3

    说明

    对于样例，一种赶路方案为 2→4→5→72 \rightarrow4 \rightarrow 5 \rightarrow 72→4→5→7，总共需要 333 天。

    T3. 村长分钱【算法赛】

    问题描述

    那天，宋江在梁山聚义厅中，与众兄弟们正饮酒作乐。喝得正高兴时，武松大步走了进来，手里还提着大碗酒，爽朗说道：“哥哥们！我今日在山下小村遇到一件奇怪的事儿。”

    宋江见状，笑道：“兄弟，有啥稀奇事？说来让大家也乐呵乐呵。”

    武松放下酒碗，抹了抹嘴巴，道：“我在村子里碰到村长，他拿着一袋银子，愁眉苦脸。说是想把这些银子分给村民，但不知道每人分多少银两合适。”

    李逵一听，咧嘴大笑：“嘿！这有啥好犯愁的？村长啥事儿都要愁，难不成是怕银两分不出去？”

    武松摇了摇头，解释道：“村里的人口众多，压根不用担心银两分不出去。可是村长有个特别的要求，他希望这些银两能够平均地分给若干村民，每次分的银两数量相同，直到袋里剩下的银两不足以再分一次为止。关键是，他希望最后剩下的银子正好是 kkk 两，好买点酒喝。”

    “这事简单！咱们就设每次分的银两数为xxx，只要满足 nnn 除以 xxx 后的余数为 kkk，不就找到了一个合适的 xxx、确定了每人分多少银两合适了嘛？！”

    “对，对的！兄弟所言极是。这样吧，咱大伙一起来来算算，一共有多少个不同的 xxx，满足 nnn 除以 xxx 的余数为 kkk。” 宋江提议道。

    众好汉们纷纷点头。

    现在，请你和众好汉一起，算算满足条件的 xxx 的个数。注意，xxx 必须是正整数，且 xxx 不能大于 nnn。

    输入格式

    输入一行，包含两个整数 nnn（1≤n≤1091\leq n \leq 10^91≤n≤109） 和 kkk（0≤k<n0 \leq k < n0≤k<n），其中 nnn 为村长手中的银子总数，kkk 是希望剩下的银子数量。

    输出格式

    输出一个整数，表示满足条件的不同的 xxx 的个数。

    样例输入

    13 3

    样例输出

    2

    样例说明

    满足条件的 xxx 有 555、101010，共 222 个。

    T5. 好汉身份【算法赛】

    问题描述

    当梁山迎来了新的一批好汉加入时，宋江哥哥决定要给各位兄弟改善一下伙食，便派了“智多星”吴用下山采购。

    吴用来到山下一看，好巧不巧，正好赶上一年一度的梁山跳蚤市场开市。他们发现市场上共有 2N2N2N 件商品，这第 iii 件商品，若是寻常百姓购买，需花费 aia_iai​ 两银子；可若是梁山好汉亮出身份，那店家便会给出一个“梁山好汉价”，需花费 bib_ibi​ 两银子。

    回到山上，吴用将此事告知了宋江。宋江听罢，哈哈大笑，说道：“妙啊！咱们来玩个游戏！我扮作寻常百姓，吴军师你亮出身份，咱们比试一番，看看最后谁花的银子少！”

    于是，宋江摇身一变，换上了一身寻常百姓的衣裳，只带了一顶普通的帽子，遮住了他那标志性的红脸膛，便大摇大摆地混进了熙熙攘攘的跳蚤市场。而吴学究则摇着羽扇，施施然地来到了市场，准备和宋江来一场斗智斗勇的比拼。

    他们定下规矩：双方轮流购买商品，宋江先出手，吴用后出手，直到所有商品都被买走为止。设宋江最终花费了 XXX 两银子，而吴用最终花费了 YYY 两银子。宋江试图最小化 X−YX - YX−Y，而吴用试图最大化 X−YX - YX−Y。

    现在，请你计算，当双方都采取最优策略时，X−YX - YX−Y 的值会是多少。

    输入格式

    第一行一个整数 nnn（1≤n≤1031\leq n \leq 10^31≤n≤103），表示共有 2n2n2n 件商品。

    第二行 2n2n2n 个整数，a1,a2,⋯ ,a2na_1,a_2,\cdots,a_{2n}a1​,a2​,⋯,a2n​（1≤ai≤1091\leq a_i \leq 10^91≤ai​≤109），其中 aia_iai​ 表示第 iii 件商品的“寻常百姓价”。

    第三行 2n2n2n 个整数，b1,b2,⋯ ,b2nb_1,b_2,\cdots,b_{2n}b1​,b2​,⋯,b2n​（1≤bi≤1091\leq b_i \leq 10^91≤bi​≤109），其中 bib_ibi​ 表示第 iii 件商品的“梁山好汉价”。

    输出格式

    输出一行一个整数，表示答案。

    样例输入

    1
3 1
4 2

    样例输出

    -3

    样例说明

    宋江可以先手选择第 222 件商品，花费 111 两银子。此时吴用只能选择第 111 件商品，花费 444 两银子。两者的差值为 −3-3−3。

    T6. 武功秘籍【算法赛】

    问题描述

    神行太保戴宗获得了一本武功秘籍，据说练成之后可以日行千里，夜走八百。戴宗正想找个僻静的地方修炼一番，却发现修炼它需要用到一种特殊的数字，叫做“梁山数”。这“梁山数”，和平常的数字不太一样，它的每个数位上的数字都有个上限，超过了就不行。比如，个位数上限是 555，十位数上限是 333，百位上限是 111，那么最大的三位“梁山数”就是 135135135。

    想修炼好这门神功，就需要先找到第 kkk 小的“梁山数”。但是，这可难倒了戴宗，于是，他急忙找到浪子燕青，将事情的来龙去脉告诉了他，并请他帮忙计算出第 kkk 小的“梁山数”。

    燕青看了看，心想这“梁山数”虽然奇怪，但也不过是换了一种计数方式而已，本质上还是数字，于是便答应下来。戴宗给了燕青每个数位上的上限值，用一个整数 nnn 表示，nnn 的第 iii 位数字 nin_ini​ 表示“梁山数”从右往左数第 iii 位上的最大值。

    现在，请你协助燕青，解决这个问题。

    输入格式

    输入一行，包含两个整数 n,kn,kn,k（1≤n,k≤10181\leq n,k \leq 10^{18}1≤n,k≤1018），分别表示每个数位的最大值、需要找的“梁山数”的序号。

    保证数据合法。

    输出格式

    输出一个整数，表示第 kkk 小的“梁山数”。

    样例输入

    135 8

    样例输出

    11

    样例说明

    当 n=135n = 135n=135 时，所有的‘梁山数’从小到大排列是：0,1,2,3,4,5,10,11,12,13,14,15,20,21,...0, 1, 2, 3, 4, 5, 10, 11, 12, 13, 14, 15, 20, 21, ...0,1,2,3,4,5,10,11,12,13,14,15,20,21,...，其中第 888 小的是 111111。

    T7. 朝廷查账【算法赛】

    问题描述

    “哥哥！不好啦！出大事啦！”

    只见小弟慌慌张张地跑进聚义厅，满头大汗，上气不接下气。

    “军师，你的意思是，只要用这两种方法，就能把账目 AAA 调整成和账目 BBB 一模一样吗？”

    “不一定，我得仔细算算才能知道。”

    现在，请你帮军师算一算，经过若干次（可以是 000 次）的调整，这两本账目能不能一模一样。

    输入格式

    第一行包含一个整数 ttt（1≤t≤1031 \le t \le 10^31≤t≤103），表示测试数据的组数。

    接下来是 ttt 组测试数据，每组数据的格式如下：

    • 第一行包含两个整数 aaa 和 bbb（1≤a,b≤1051 \le a, b \le 10^51≤a,b≤105），分别表示账目 AAA 和 BBB 中已经记录的数的个数。
    • 第二行包含 aaa 个整数 A1,A2,…,AaA_1, A_2, \dots , A_aA1​,A2​,…,Aa​（1≤Ai≤1091\leq A_i \leq 10^91≤Ai​≤109），表示账目 AAA 中前 aaa 笔记录的数。
    • 第三行包含 bbb 个整数 B1,B2,…,BbB_1, B_2, \dots, B_bB1​,B2​,…,Bb​（1≤Bi≤1091\leq B_i \leq 10^91≤Bi​≤109），表示账目 BBB 中前 bbb 笔记录的数。

    保证在所有的测试数据中，aaa 和 bbb 的总和不超过 2×1052\times 10^52×105。

    输出格式

    对于每组测试数据，如果能将账目 AAA 调整成和账目 BBB 一致，则输出 YES，否则输出 NO。

    样例输入

    2
2 2
1 1
1 2
2 1
2 4
3

    样例输出

    YES
NO

    样例说明

    对于第一组数据，起初：

    A=[1,1,0,0,0,⋯ ]A = [1, 1, 0, 0, 0, \cdots] A=[1,1,0,0,0,⋯]

    选择 r=3,q=1,p=2r = 3, q = 1,p =2r=3,q=1,p=2，令 A3=A1+A2A_3 = A_1 + A_2A3​=A1​+A2​：

    A=[1,1,1,0,0,⋯ ]A = [1, 1, 1, 0, 0, \cdots] A=[1,1,1,0,0,⋯]

    T8. 晁盖分饼【算法赛】

    问题描述

    在梁山泊的一次盛大聚会上，晁盖为了庆祝诸位英雄好汉的勇猛，特意准备了一块巨大的圆形烧饼。这块烧饼需要被分配给参加聚会的 kkk 名好汉，每位好汉都应分得一份。

    晁盖大手一挥，对着各位兄弟说道：“兄弟们，今天这块烧饼，咱们要分而食之，以庆祝咱们的义气！”

    李逵一听，顿时来了兴致，嚷嚷道：“哥哥，这烧饼怎么分？俺要吃最大的那块！”

    晁盖笑着说：“放心，少不了你的。咱们按规矩来，每个人分到的烧饼份额，都用一个整数来表示，这个整数至少是 111，最大不超过 nnn。比方说，如果谁的烧饼份额为 222，那么他将会分得整张烧饼的 12\frac{1}{2}21​；如果谁的烧饼份额为 333，那么他将会分得整张烧饼的 13\frac{1}{3}31​！”

    吴用摇着羽扇，补充道：“哥哥，这烧饼得正好分完才行！如果只有两个人分烧饼，一个人分到了 13\frac{1}{3}31​ 的烧饼，另一个人分到了 12\frac{1}{2}21​ 的烧饼，那这烧饼就还剩下 16\frac{1}{6}61​ 没分完，这就不好了，咱可不能浪费啊！”

    晁盖点点头：“军师说得对，这饼一定要正正好好的分完，不能有剩下的。现在，我想问问各位兄弟，你们谁能算出来，有多少种不同的分饼的方法，能够把这块饼正好分给在座的 kkk 位兄弟，且每个人分到的饼的份额都是 111 到 nnn 之间的整数？”

    不同的分饼方法定义为：如果在两种分饼方法中，至少有一个好汉获得的份额不同，则视为不同的分饼方法。

    • [2,4,4][2,4,4][2,4,4]：第一个好汉分得整张烧饼的 12\dfrac{1}{2}21​，第一个好汉分得整张烧饼的 14\dfrac{1}{4}41​，第三个好汉分得整张烧饼的 14\dfrac{1}{4}41​。

    “哥哥，这也太难算了吧！”好汉们纷纷摇头。

    “哈哈，不用担心，我已经算过了，答案不会超过 2602^{60}260 种！”晁盖笑着说。

    请你算算有多少种分配烧饼的方式。

    输入格式

    T9. 酒量争霸赛【算法赛】

    问题描述

    话说在梁山水泊，宋江大哥为了让兄弟们在酒桌上比出个高下，决定来一场“酒量争霸赛”。这场比赛不比拳脚功夫，只比谁能喝得多，喝得久。

    于是，nnn 个好汉齐聚一堂，各个摩拳擦掌，准备展示自己的酒量。宋江大哥发话：“每位兄弟上场前，按照顺序报报自己的酒量，让大家心里有个数。” 于是，兄弟们依次按顺序报上了自己的初始酒量：a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​。

    接着，宋江大哥详细讲起了比赛的规则：

    1. 每一轮开始前，所有兄弟先计算一下“累计酒量”，也就是这轮每个人要喝的酒量。具体来说，第 iii 个人这轮要喝的酒量 bib_ibi​ 是前面所有人的酒量之和：

      bi=a1+a2+⋯+aib_i = a_1 + a_2 + \cdots + a_i bi​=a1​+a2​+⋯+ai​

      这就意味着，越靠后的兄弟，这轮得喝的酒越多！

    2. 喝完这一轮后，最靠前的的那位兄弟会被淘汰出局（即序列 bbb 的第一个元素）。

    3. 剩下的兄弟们继续喝下一轮。在下一轮中，大家的酒量数据在每轮结束后会更新成上一轮的累计酒量（即上一轮计算出来的累计酒量 bbb 会成为下一轮的初始酒量 aaa）。接着，大家再重新计算各自这一轮要喝的酒量。

    4. 如此这般，每轮淘汰掉一个兄弟，直到最后只剩下一位能喝的“酒霸”。

    宋江大哥笑着说：“这位酒霸的最终酒量，也不知道会是多少。”

    对此，请你计算，经过一轮轮“喝酒淘汰赛”，最终那位唯一的酒霸，他的酒量对 109+710^9 + 7109+7 取余会是多少呢？

    输入格式

    第一行包含一个整数 nnn (1≤n≤105)(1 \leq n \leq 10^5)(1≤n≤105)，表示好汉的数量。

    第二行包含 nnn 个整数 a1,a2,…,ana_1, a_2, \ldots, a_na1​,a2​,…,an​ (1≤ai≤109)(1 \leq a_i \leq 10^9)(1≤ai​≤109)，表示每位好汉的初始酒量。

    输出格式

    输出一行，表示最终剩下的酒霸的酒量对 109+710^9 + 7109+7 取余的结果。

    样例输入

    3
1 2 3

    样例输出

    9

    样例说明

    1. 第一轮：
      • 初始酒量：a=[1,2,3]a = [1, 2, 3]a=[1,2,3]
      • 累计酒量：b=[1,3,6]b = [1, 3, 6]b=[1,3,6]
      • 淘汰第一个：111
    2. 第二轮：
      • 更新酒量：a=[3,6]a = [3, 6]a=[3,6]
      • 累计酒量：b=[3,9]b = [3, 9]b=[3,9]
      • 淘汰第一个：333
    3. 第三轮：
      • 更新酒量：a=[9]a = [9]a=[9]
      • 只有一人，结束。

    最终酒霸的酒量为 999，输出结果为 999。
